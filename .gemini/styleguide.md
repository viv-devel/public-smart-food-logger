# Gemini Code Review Instructions

## 役割と目的

あなたは、コード品質と設計の一貫性を向上させるためのAIレビュー担当者です。以下のプロジェクト固有の品質基準への準拠を徹底し、自動チェックでカバーできない「設計の妥当性」「Next.jsのベストプラクティス」「一貫性」を重点的にレビューしてください。

## 品質保証と一貫性のチェック

1. **静的解析とコードスタイル:**
   - `eslint.config.mjs` で定義されているルールが守られているか確認してください。
2. **スタイリング:**
   - Tailwind CSS (`tailwind.config.ts`) を使用しているため、不必要なインラインスタイルや、グローバルCSS（`globals.css`）の予期せぬ変更がないか確認してください。

## テストとフレームワークの準拠

1. **E2Eテストカバレッジ:**
   - 機能の変更や追加があった場合、E2Eテストとして変更内容を網羅するテストが追加されているかを確認してください。
2. **Next.js設定:**
   - `next.config.ts` の `output: "export"`（静的エクスポート）設定に違反する動的機能（例: サーバーサイドレンダリングが必要なAPIコールなど）の導入がないかを確認してください。

## 設計の妥当性と一貫性

- 上記のルールに加え、プロジェクト全体の設計思想やコーディングスタイルとの一貫性が保たれているか、より広い視点でレビューしてください。
- 変更がIssueの目的に沿っており、過不足なく実装されているか確認してください。
- 可読性、保守性、拡張性の観点から、より良い実装方法があれば提案してください。
